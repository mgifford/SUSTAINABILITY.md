Machine-Readable Metadata: Working Demo

This page demonstrates machine-readable metadata patterns as required by WSG 4.4 — Structure Your Metadata for Machine Readability. It combines JSON-LD (in <head>), RDFa (on this <article> element), and essential <meta> tags to show how each approach works.

Last modified: 11 March 2026.

1. What this page demonstrates

  • JSON-LD blocks in <head> — three blocks covering the article, a FAQ, and breadcrumbs. These are the primary structured data signals for search engines and AI crawlers.
  • RDFa attributes on the <article> element — shows how visible content can carry metadata inline using vocab, typeof, and property.
  • Microdata on an example content block — shows the itemscope, itemtype, and itemprop pattern for completeness.
  • Canonical URL and Open Graph tags — in <head>, used by social platforms and link-preview scrapers.

3. Sitemap.xml pattern

A sitemap.xml file lists URLs and when they were last modified. Crawlers use <lastmod> to skip pages that have not changed, reducing unnecessary server requests. 

<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://example.org/</loc>
    <lastmod>2026-03-11</lastmod>
    <changefreq>monthly</changefreq>
    <priority>1.0</priority>
  </url>
</urlset>

Declare the sitemap URL in robots.txt: 

Sitemap: https://example.org/sitemap.xml

4. FAQ section (structured for rich results)

The questions below are also described by the FAQPage JSON-LD block in <head>. This enables search engines to show answers directly in results, and AI assistants to answer without loading the page.

Why does machine-readable metadata reduce energy use?
Structured metadata lets search engines and AI assistants answer user questions directly from the metadata, without loading the full page. A JSON-LD block is typically under 5 KB, compared with 100–3,000 KB for a rendered page. Fewer full page loads means less bandwidth, less server compute, and less client-side energy.
Which structured data format should I use?
Start with JSON-LD. It is decoupled from visible markup, easy to test as standalone JSON, and supported by all major search engines. Add sitemap.xml generation early — it is nearly free with static site generators and tells crawlers which pages to skip when content has not changed.
What is the difference between RDFa and Microdata?
Both annotate visible HTML content with Schema.org vocabulary. RDFa uses vocab, typeof, and property attributes; Microdata uses itemscope, itemtype, and itemprop. Both are supported by major search engines, but JSON-LD is generally preferred for new implementations because it is easier to maintain.
