Semantic HTML Patterns: Working Demo

This page demonstrates semantic HTML as required by WSG 3.7. It is validated automatically by html-validate in CI.

1. Landmark regions

Native sectioning elements (<header>, <nav>, <main>, <aside>, <footer>) provide landmark roles without extra ARIA attributes.

2. Interactive controls

Use <button> for actions and <a> for navigation. Never use <div onclick> as an interactive control.

3. Form labels and inputs

Every input must have an explicit <label> linked via the for/id pair. Do not rely on placeholder text as a label.

4. Images

Informative images need descriptive alt text. Decorative images use alt="".

5. Lists and tabular data

Use lists for enumerable items, tables for relational data.

  1. Install html-validate: npm install --save-dev html-validate
  2. Add .htmlvalidate.json to the project root
  3. Run npx html-validate "**/*.html"
  4. Add the CI job to enforce rules on every pull request
Tool comparison: html-validate vs HTMLHint vs Nu Checker
Tool Focus Network required
html-validate Semantic correctness, ARIA misuse No
HTMLHint Authoring errors, alt text, IDs No
Nu Checker (vnu) Full HTML5 spec conformance Optional (Docker/jar)
